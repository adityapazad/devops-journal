# Services & systemd

## Services
A service is a background process that runs continuously and provides functionality to the system or other programs.

## systemd
systemd is a service manager and initializaton system in modern linux distributions.

## Commands
- `systemctl` : a CLI tool to interact with systemd
- `systemctl status` : checks current status of a service
- `systemctl start` : starts a stopped service
- `systemctl stop` : stops a running service
- `systemctl restart` : stops and starts a service again
- `systemctl enable` : enable the service to start automatically at boot
- `systemctl disable` : prevents service to start at boot
