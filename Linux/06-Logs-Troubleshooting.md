# Logs & Troubleshooting

## Logs
Logs are timestamped records of system/services events. Logs are used for debugging, monitoring and auditing.
`/var/logs` is a directory where Linux stores log files.

## Commands
- `journalctl` : used to read logs from systemd journal
- `tail` : shows last lines of a file
- `-f` : follows the file in real time
- `tail -f` : used for live monitoring of logs
